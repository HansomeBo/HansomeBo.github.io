    CPU到100%分析
    2020/08/20 基础

    本质

    一定有某个线程 CPU 占用比较高
    分业务线程和系统线程

    步骤

    1. top -c 显示进程列表，输入 P 根据 cpu 使用率排序，查看 cpu 占用比较高的进程
    2. top -Hp 进程 pid ，输入 P 根据 cpu 使用率排序，查看 cpu 占用比较高的线程
    3. printf “%x\n” 线程 pid ，将线程 pid 转换成 16 进制
    4. jstack 进程 pid | grep ‘线程 pid’ 找到这个线程的堆栈信息，定位对应的代码

    jmap
    jinfo
    Java VisualVM
